一、SSRF漏洞

漏洞简介

SSRF 漏洞(服务器端请求伪造)是一种由攻击者构造请求包，服务器接收到请求后由服务端发起请求的一个安全漏洞；一般情况下是由于服务器存在发送请求的功能，例如读取远程服务器文件、获取内部服务器信息等功能点。

攻击者可以利用该漏洞向外部网络发起攻击，例如攻击内部网络、访问外部系统和获取敏感数据等。该漏洞通常存在于 web 应用程序中，由于应用程序对用户输入未进行充分的验证或过滤，攻击者可以构造恶意请求，使应用程序将请求发送到攻击者指定的远程服务器或应用程序。一般情况下，SSRF攻击的目标是从外网无法访问的内部系统。

漏洞原理

SSRF的形成大多是由于服务端提供了从其他服务器应用获取数据的功能且没有对目标地址做过滤与限制。例如，黑客操作服务端从指定URL地址获取网页文本内容，加载指定地址的图片等，利用的是服务端的请求伪造。SSRF利用存在缺陷的Web

应用作为代理攻击远程和本地的服务器。

主要攻击方式如下所示。

  • 对外网、服务器所在内网、本地进行端口扫描，获取一些服务的banner信息。
  • 攻击运行在内网或本地的应用程序。
  • 对内网Web应用进行指纹识别，识别企业内部的资产信息。
  • 攻击内外网的Web应用，主要是使用HTTP GET请求就可以实现的攻击(比如struts2、SQli等)。
  • 利用file协议读取本地文件等。

漏洞修复

SSRF漏洞修复建议：

  1. 验证和过滤用户输入：应用程序应该对用户输入的 URL、IP 地址等参数进行充分的验证和过滤，确保用户输入的参数符合应用程序的预期。例如，可以使用正则表达式或白名单过滤来验证 URL，以确保请求只能发送到允许的地址。
  2. 限制应用程序访问外部网络的权限：应用程序应该限制应用程序访问外部网络的权限，例如使用白名单、限制端口和协议等方式，以减少攻击面。例如，可以使用网络安全组或防火墙来限制应用程序的访问权限，以防止应用程序将请求发送到不受信任的主机。

4、漏洞修复

SSRF的修复比较复杂，需要根据业务实际场景来采取不同的方案，从代码层来修复，需要注意以下几点：

  1. 去除url中的特殊字符

  2. 判断是否属于内网ip

  3. 如果是域名的话，将url中的域名改为ip

  4. 请求时设置host header为ip

  5. 不跟随30x跳转
